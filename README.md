# Third Party Risk Management (TPRM) Portfolio

## Objective
This project simulates the real scenarios of an end-to-end Third Party Risk Management lifecycle for a financial services organization.

## Scope
- Vendor onboarding
- Risk assessment
- Due diligence
- Control validation
- Ongoing monitoring
- Issue remediation
- Dashboard reporting

## Methodology
Risk scoring is based on Impact x Likelihood model.
Vendors are categorized as Critical, High, Medium, or Low.

## Tools Used
- Microsoft Excel
- Risk Scoring Model
- Sample Compliance Mapping (SOC 2, ISO 27001)
